# Dead Lock

한 번에 한개의 프로세스만이 공유 자원을 사용할 수 있어야 됨 : 상호배제 (Mutual exclusion)
공유 자원과 공유자원을 사용하기 위해 대기하는 프로세스들이 원형으로 구성되어 자신에게 할당된 자원을 점유하면서 앞 뒤에있는 프로세스의 자원을 요구해야 함 : 환형대기 Circular Wait 

상호배제에 의해 나타나는 문제점으로 둘 이상의 프로세스들이 자원을 점유한 상태. 

서로다른 프로세스가 점유하고있는 자원을 요구해서 무한정 기다리는 현상 


상호배제: 한 번에 한개 의 프로세스만 공유자원
점유와 대기: 하나 점유하고있으면서 다른 프로세스에 할당되어 사용되고 있는 자원을 추가로 점유하기 위해 대기하는 프로세스가 있어야함
비선점: 다른 프로세스에 할당된 자원은 사용이 끝날 때까지 강제로 빼앗을 수 없어야함
환형 대기: 원형으로구성 자신에게 할당된 자원을 점유하면서 앞이나 뒤에있는 프로세스의 자원을 요구해야 함


- 교착 상태의 해결 방법

예방 기법 : 사전에 시스템을 제어하는 방법 => 자원낭비가 가장 심함 

회피 기법 : Banker's 알고리즘 / 은행에서 모든 고객의 요구가 충족되도록 현금을 할당하는 데서 유래한 기법 / 교착상태가 발생하면 적절히 피해나가는 방법 / 교착이 발생할 가능성을 배제하지않음

발견 기법 : 시스템에 교착 상태가 발생했는지 점검하여 교착상태에 있는 프로세스와 자원을 발견하는거  

회복 기법 :교착을 일으킨 애를 종료하거나 프로세스에 할당된 자원을 선점해서 프로세스나 자원을 회복하는것
