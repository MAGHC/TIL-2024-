관계해석: 관계 데이터의 연산을 표현하는 방법으로, 관계 데이터 모델의 제안자인 코드 가 수학의 술어해석에 기반을 두고 관계 데이터베이스를 위해 제안함 

후보키 : 릴레이션을 구성하는 속성들 중에서 튜플을 유일하게 식별하기 위해 사용하는 속성들의 부분집함 /기본키로 사용할 수 있는 속성들을 의미함 / 유일성과 최소성을 만족 
(candidate key)
기본키: 후보키 중에서 특별히 선정된 메인키로 중복 불가 
(primary key)
대체키 : 후보키가 둘 이상일 때 기본키를 제외한 나머지 후보키를 의미 보조키라고도함. 
(alternative key)

유일성: 하나의 키 값으로 하나의 튜플만을 유일하게 식별할수 있어야 함 
최소성: 모든 레코드들을 유일하게 식별하는데 꼭 필요한 속성으로만 구성되어야 함

친구들끼리 서로서로 구분할때는 이름을 쓰지만 이름은 기본키가될수없음 (중복불가) 


슈퍼키 : 한 릴레이션 내에 있는 속성들의 집함으로 구성되는 키로 릴레이션을 구성하는 모든 튜플에 대해 유일성은 만족시키지만 최소성은 만족 못 시킴
외래키 : 다른 릴레이션의 기본키를 참조하는 속성 또는속성들의 집합

개체 무결성: 기본 테이블의 기본키를 구성하는 어떤 속성도 null값이나 중복값을 가질 수 없다는 규정

참조 무결성: 외래키 값은 null 이거나 참조 릴레이션의 기본키 값과 동일해야 함. 즉 릴레이션은 참조할 수 없는 외래키 값을 가질 수 없다는 규정

침입 탐지 시스템, 방지 시스템 , 데이터 유출방지 방화벽등

방화벽: 기업이나 조직 내부의 네트워크와 인터넷간의 전송되는 정보를 선별해서 수용/ 거부 / 수정 기능을 가진 침입 차단 시스템 / 내부에서 외부는 그냥 통과시키고 외부에서 내부로 들어오는것만 엄밀하게 체크해서 통과 

침입 탐지 시스템: 컴퓨터 시스템의 비정상적인 사용/오용/남용 등을 실시간 탐지 시스템 

침입 방지 시스템 (IPS): 방화벽이랑 침입 탐지 시스템을 결합한 것으로 비정상적인 트래픽 능동적으로 차단하고 격리하는 등의 방어 조취를 취하는 보안 솔루션 

데이터 유출 방지(DLP) 사내 직원이 사용하는 PC와 네트워크 상의 모든 정보를 검색하고 메일 메신저 웹하드 네트워크 프린터 등의 사용자 행위를 탐지 및 통제하여 내부정보가 외부로 유출되지않도록 사전에 방지함.

웹앱방화벽 (WAF) 와프 :  일반 방화벽이 탐지못하는 SQL삽임, XSS 등의 웹 기반 공격을 방어할 목적으로 만들어진 웹서버에 특화된 방화벽 

SQL삽입 : 봇넷이나 전문 스캐너프로그램으로 취약한 사이트를 발견하면 무차별 공격 
XSS : 스크립트의 취약점을 악용한 해킹 기법 

VPN: 가상 사설 통신망 
통신 사업자의 공중 네트워크와 암호화 기술을 이용하여 사용자가 마치 자신의 전용 회선을 사용하는 것처럼 해주는 보안 솔루션 
암호화된 규격을 통해 인터넷망을 전용선의 사설망을구축한 것처럼 이용할 수 있음. 



NAC: 나크 네트워크 엑세스 컨트롤 : 네트워크에 접속하는 내부 PC의 맥주소를 IP관리 시스템에 등록하고 일관된 보안 관리 기능을 제공함.

내부 PC의 소프트웨어 사용 현황을 관리하여 불법적인 소프트웨어 설치를 방지 

SIEM : SIM+SEM (앞에껀 보안정보관리 뒤에껀 보안이벤트관리) 방화벽 IDS IPS 웹 방화벽 VPN등에서 발생한 로그 및 보안 이벤트를 통합 및 연동하여 종합적인 보안 관리 체계를 수립 할 수 있음. 

ESM : SIEM과 기능은 동일하지만 짧은 기간의 로그 및 보안 이벤트를 관리하는 보안 솔루션 


# 교착 상태 

데드락  

상호 배제에 의해 나타나는 문제점.  서로다른애가점유하고있는 요구해서 무한정 기다리는거 

로브트 상하체가있는데 형은 상체 동생은 하체 서로 달라고 하고 합의안나는 형태

교착상태의 필요 충분 조건 한개라도 충족안되면 교착안됨 

상호배제 / 점유대기 / 비선점 / 환형대기 

상호배제? => 한번에하나의 프로세스만 공유자원을 사용할 수 있어야됨 
점유와 대기? => 최소한 하나를 점유하면서 다른애를 추가로 점유하기위해 대기하는 애가있어야됨 
비선점? => 할당되면 다른애가 강제로 못 뺏음 
환형대기? => 공유자원과 자원을 사용하기 위해 대기하는 프로세스들이 원형으로 구성되어있어 자신에게 할당된 자원을 점유하면서 앞이나 뒤에있는 프로세스의 자원을 요구해야 함 

해결방법

예방 기법 : 사전에 시스템을 제어하는 방법 / 네가지 발생 조건중 하나를 제거함 (자원의 낭비가 가장 심함)
회피 기법 : 교착상태가 발생할 가능성을 배제하지않고 발생하면 피해나감 (Banker's algorithm) 이 사용됨 은행원알고리즘 은행에서 모든 고객의 요구가 충족되도록 현금을 할당하는 데서 유래한 기법 
발견 기법 : 시스템에 교착상태가 발생했는지 점검하여 프로세스와 자원을 발견하는 것
회복 기법 : 교착 상태를 일으킨 프로세스를 걍 종료 하거나 할당된 애를 선점해서 회복시킴
